Skills Basics

SKILL.md structure and slash commands

Not Started

What are Skills?

Skills teach Claude how to perform specific tasks. They can be automatically discovered based on context or invoked directly with slash commands like /skill-name.

SKILL.md Structure

Skills are defined as SKILL.md files with YAML frontmatter:

.claude/skills/my-skill/SKILL.md
---
name: my-skill
description: What this skill does (used by Claude for discovery)
allowed-tools: [Bash, Read, Write]  # optional
model: sonnet                        # optional
user-invocable: true                 # enables /skill-name
---

# Skill Instructions
Explain how Claude should perform this task.

## How it works
1. First step...
2. Second step...
3. Output results...

Built-in Slash Commands

/commit

Auto-generate commit message from staged changes and commit.

/review-pr

Review a pull request and provide feedback.

/init

Initialize CLAUDE.md with project context.

/compact

Summarize conversation to save context window.

Custom Skills Creation

Create your own skills and skill discovery

Not Started

Creating a Custom Skill

Example: Create a skill for writing survey documents.

.claude/skills/survey-writer/SKILL.md
---
name: survey-writer
description: Write academic survey documents from paper summaries
allowed-tools: [Read, Write, WebFetch]
user-invocable: true
---

# Survey Writer Skill

You are an expert at writing academic survey documents.

## Process
1. Read the provided paper summaries
2. Identify common themes and categorize papers
3. Write an introduction explaining the research area
4. Create sections for each category
5. Write a conclusion with insights and future directions

## Output Format
- Use markdown format
- Include proper citations
- Create a clear table of contents

Skill Locations & Priority

  1. 1. Project: .claude/skills/<name>/SKILL.md (highest priority)
  2. 2. User: ~/.claude/skills/<name>/SKILL.md
  3. 3. Global: Skills from installed plugins

Skill Discovery

Claude automatically discovers skills based on the description field. When you ask Claude to do something matching a skill's description, it will use that skill.

Example
# If you have a skill with description: "Write academic survey documents"
# Claude will automatically use it when you say:

User: "Write a survey document from these paper summaries"
# Claude: [Uses survey-writer skill automatically]

# Or invoke directly:
User: "/survey-writer"

MCP Basics

Model Context Protocol concept and server connection

Not Started

Model Context Protocol (MCP)

MCP is an open protocol that standardizes how AI assistants connect to external data sources and tools. Think of it as a "USB for AI" - a universal way to plug in capabilities.

Why MCP Matters

Before MCP, every AI integration required custom code. MCP provides a standard interface for:

  • Tools - Functions the AI can call (search, database queries, API calls)
  • Resources - Data the AI can read (files, databases, web content)
  • Prompts - Pre-defined prompt templates for specific tasks

Adding an MCP Server

Terminal
# Add MCP server using claude mcp add command
claude mcp add semantic-scholar

# Or add with specific package
claude mcp add semantic-scholar -- npx -y @anthropic-ai/mcp-server-semantic-scholar

# View connected MCP servers
/mcp

Semantic Scholar MCP

Academic paper search and retrieval

Not Started

Setting Up Semantic Scholar MCP

.claude/settings.local.json
{
  "mcpServers": {
    "semantic-scholar": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-semantic-scholar"],
      "env": {
        "SEMANTIC_SCHOLAR_API_KEY": "${SEMANTIC_SCHOLAR_API_KEY}"
      }
    }
  }
}

Available Tools

search_papers

Search for papers by keywords, get titles, abstracts, and citations.

get_paper_details

Get full details of a specific paper by ID.

get_citations

Get papers that cite a specific paper.

get_references

Get papers referenced by a specific paper.

Usage Example

Claude Conversation
User: "Search for papers about large language model alignment"

Claude: [Uses semantic-scholar.search_papers tool]
# Returns: List of papers with titles, authors, abstracts

User: "Get details and citations for the first paper"

Claude: [Uses semantic-scholar.get_paper_details]
Claude: [Uses semantic-scholar.get_citations]
# Returns: Full paper details and citing papers
Warning

API Key Security

Store your API key as an environment variable. Never commit settings.local.json to git.

Skills + MCP Workflow

Combining skills and MCP for powerful automation

Not Started

Power of Combination

Skills define HOW to do tasks, MCP provides WHAT tools to use. Combined, they create powerful automated workflows.

Paper Survey Workflow Example

Workflow
# 1. MCP: Search papers
Claude uses semantic-scholar.search_papers("LLM alignment")
  -> Returns 50 papers

# 2. MCP: Get details for top 10
Claude uses semantic-scholar.get_paper_details for each
  -> Returns abstracts, citations, authors

# 3. Skill: Summarize papers
/survey-writer skill processes the data
  -> Creates structured summaries

# 4. Skill: Write survey document
Skill creates survey.md with:
  - Introduction
  - Categorized papers
  - Analysis and insights
  - References (BibTeX)

Using Subagents for Parallel Work

Use subagents to parallelize paper analysis:

Parallel Paper Analysis
User: "Analyze these 10 papers in parallel and summarize each"

# Claude spawns multiple agents:
Agent 1 (Explore): Analyze paper 1-2
Agent 2 (Explore): Analyze paper 3-4
Agent 3 (Explore): Analyze paper 5-6
Agent 4 (Explore): Analyze paper 7-8
Agent 5 (Explore): Analyze paper 9-10

# All run simultaneously, then results are combined

Complete Survey Project Structure

Project Structure
paper-survey/
+-- .claude/
|   +-- settings.local.json    # MCP: Semantic Scholar
|   +-- skills/
|       +-- survey-writer/
|           +-- SKILL.md       # Custom survey skill
+-- survey.md                  # Output: Survey document
+-- references.bib             # Output: BibTeX references
+-- CLAUDE.md                  # Project context

TERMINOLOGY

Skills

Instructions that teach Claude specific tasks. Defined in SKILL.md files with frontmatter for metadata. Invoked with /skill-name or discovered automatically.

MCP (Model Context Protocol)

An open protocol for connecting AI assistants to external tools and data sources. MCP servers provide tools, resources, and prompts that Claude can use.

Slash Commands

Commands starting with / that invoke specific functionality. Built-in commands include /commit, /init, /compact. Custom skills can also be invoked this way.

Skill Discovery

Claude automatically finds and uses relevant skills based on their descriptions. When a request matches a skill's description, Claude applies that skill without explicit invocation.
